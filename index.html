Connection Manager

/ tɛŋɪŋɡarˈstjouːrɪ / — Icelandic

A nerdy, opinionated SSH connection manager that stores your connections in ~/.ssh/config.tengingarstjori and leaves your existing config completely untouched.

What it does

01
Non-invasive config

One Include line added. All managed connections live in their own file — remove it any time, nothing breaks.

02
ProxyJump support

Bastion hosts and multi-hop chains with one flag. Custom users, non-standard ports, deep tunnels.

03
Port forwarding

Local and remote forwards, multiple tunnels per connection. Comma-separate them. Old syntax auto-corrected.

04
Flexible list views

Table, compact, or JSON. Filter by tag, search by name or notes, sort by last used. Compose freely.

05
Export & import

Backup as JSON or raw SSH config. Strip key paths for safe sharing. Import with skip, overwrite, or rename strategies.

// commands

The connection journal

Every command is designed to be typed once and remembered. Flags that make sense. Output you can read. No config language to learn.

setup

Initialize once, use forever

tg init backs up your SSH config and adds a single Include line. Everything managed by tg goes into ~/.ssh/config.tengingarstjori — a separate file you can inspect, back up, or discard independently.

tg init
$tg init
╭─ Initialize SSH Integration ────────────────╮ ╰─────────────────────────────────────────────╯ SSH config backed up to ~/.ssh/config.backup ✓ Added Include line to ~/.ssh/config ✓ Created ~/.ssh/config.tengingarstjori Ready. Run: tg add
add

Add connections your way

Flags for scripting, bare tg add for a guided prompt. Supports ProxyJump, local and remote port forwarding, SSH keys, custom ports, and notes. Everything is validated before it touches disk.

tg add
$tg add -n prod-web -h 10.0.1.10 -u deploy
✓ Added connection 'prod-web' Connect with: ssh prod-web

$tg add -n staging-db -h 10.0.2.20 -u dbadmin \
--proxy-jump bastion \
--local-forward '5432:localhost:5432'
✓ Added connection 'staging-db' Connect with: ssh staging-db
list

connect · snippet · history

Workflow commands

tg connect fires SSH and tracks usage stats. tg snippet prints the exact SSH command or config block — pipe to clipboard. tg history shows what you've been connecting to and how often.

workflow
$tg connect prod-web
↗ ssh deploy@10.0.1.10

$tg snippet staging-db
ssh -J bastion -L 5432:localhost:5432 dbadmin@10.0.2.20

$tg history
Recent connections: prod-web 2026-04-10 ×12 staging-db 2026-04-09 ×3

$tg test prod-web
✓ prod-web reachable 42ms

For the actually complex stuff

The features homelab engineers actually need — handled cleanly instead of left as an exercise for the reader.

↠
ProxyJump chains

Multi-hop through bastion hosts. Custom users, non-standard ports, chains of any length.

tg add -n deep-internal \
  --host 172.16.5.10 -u root \
  --proxy-jump "jump1.company.com,\
  user@jump2.internal.com"
⇄
Multiple port forwards

Comma-separate tunnels. Local and remote forwards in one connection. Syntax auto-corrected from legacy formats.

tg add -n dev-stack \
  --host dev.company.com -u dev \
  --local-forward "3000:localhost:3000,\
  8080:localhost:8080,\
  5432:localhost:5432"
⇅
Export & import

JSON or SSH config output. Strip key paths for safe sharing. Three conflict strategies on import.

tg export --strip-keys \
  -o safe-backup.json

tg import backup.json \
  --strategy rename

Every command

Connections
  • tg addAdd a connection (interactive or via flags)
  • tg updateUpdate fields on an existing connection
  • tg cloneDuplicate a connection with a new name
  • tg removeRemove a connection (--force skips confirmation)
  • tg showShow all fields and the generated SSH block
  • tg listList connections with filters, sorts, and formats
Workflow
  • tg connectSSH and track usage stats (--dry-run to preview)
  • tg snippetPrint the SSH command or config block
  • tg historyShow recently used connections
  • tg testCheck SSH connectivity (--all, --timeout)
  • tg exportExport to JSON or SSH config format
  • tg importImport from JSON with conflict resolution
Setup & Maintenance
  • tg initInitialize and integrate with SSH config
  • tg configManage default settings (e.g. default SSH key)
  • tg validateCheck all connections for config issues
  • tg refreshRegenerate the managed SSH config file
  • tg fix-configRepair corrupted Include lines
  • tg resetRestore original SSH config from backup
